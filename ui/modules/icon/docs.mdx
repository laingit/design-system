## Usage

Five separate SVG sprites are used to create icons — [action](/icons/#action), [custom](/icons/#custom), [doctype](/icons/#doctype), [standard ](/icons/#standard) and [utility](/icons/#utility). Link to the icon SVG sprite by targeting the icon’s hash/ID value in the use href attribute. (You can find the values on the [icon page](/icons/).)

## Primitive Icon

The icon component consumes the primitive SVG icon component. This component is purely used for displaying the SVG from the sprite sheet.

<CodeBlock toggleCode={false}>
  <svg aria-hidden="true" className="lwc-svg" title="when needed">
    <use xlinkHref="/assets/icons/standard-sprite/svg/symbols.svg#case" />

<Blockquote type="note" header="Note">

  <code className="doc">use</code> attribute with the underscore separator as shown on the icon page. For
  example: <code className="doc">log_a_call</code>.
</Blockquote>

## Accessibility

Icons require a containing element for two reasons:

- If assistive text is required, the containing element should contain both the icon and a `<span>` with hidden assistive text describing the icon using the `slds-assistive-text` class.
- If the icon is used without a visible, descriptive label, a `title` attribute is needed on the containing element. The `title` should describe the icon.

If an icon has a visible label that describes what the icon represents, no title or hidden assistive text is required.

## Icon Sizing

Icons accept a sizing class that will change the dimensions of the icon. Sizes range from `x-large` to `xx-small`. Changing the size of the icon is independent of the icons boundary size.

By default, utiity icons are 16px by 16px. For action, custom, doctype, and standard sets, the default is 24px by 24px.

|Class|Size|
|-----|----|
|`lwc-icon_x-large`|64px|
|`lwc-icon_large`|48px|
|`lwc-icon_medium`|32px|
|`lwc-icon_small` (action, custom, doctype, standard _default_)|24px|
|`lwc-icon_x-small` (utility icon _default_)|16px|
|`lwc-icon_xx-small`|12px|

## Icons with Boundary Sizing

Every icon comes with a boundary, this boundary size can be alter independently from the icon size. Sizes range from `x-large` to `xx-small`. Changing the boundary will _not_ change the size of the icon.

By default, the icon boundary is the size of the icon.

|Class|Size|
|-----|----|
|`lwc-icon-boundary_x-large`|64px|
|`lwc-icon-boundary_large`|48px|
|`lwc-icon-boundary_medium`|32px|
|`lwc-icon-boundary_small`|24px|
|`lwc-icon-boundary_x-small`|16px|
|`lwc-icon-boundary_xx-small`|12px|

## Icons with color

Icons can have their color modified by overriding the CSS custom property, `--c-theme-icon-color` inside of `:host`. We provide three colors for user feedback states such as success, warning, and error. We also provide a muted color where an icon doesn't required as much emphasis.

|Class|State|
|-----|-----|
|`lwc-icon_success`|success|
|`lwc-icon_warning`|warning|
|`lwc-icon_error`|error|

## Unset background and color

There might be situations where you want to use an icon from our Standard or Custom set but don't require the background or color. You can unset the default background and color with `lwc-icon_unset`.

After unsetting the color, the icon will inherit the font color from the app. You can apply a provide icon color of success, warning, error, or muted or override the icon color by changing the value of `--c-theme-icon-color` in your `:host`.

<Blockquote type="warning" header="Note">
  Unsetting the background and color should only be used on our custom or standard set. <strong>Usage on any other set is unsupported.</strong>
</Blockquote>
